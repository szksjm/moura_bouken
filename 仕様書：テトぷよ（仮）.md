# 仕様書：テトぷよ（仮称）

※本ドキュメントはスマホブラウザ（PWA対応）で動作する落ち物パズルゲームの初版仕様です。後続のプロトタイプ実装とプレイテスト結果を反映して随時改訂します。

---

## 1. 目的 / コンセプト
- **コンセプト**：テトリスの“形状戦略”とぷよぷよの“連鎖戦略”を融合。ブロック（以降「ピース」）は**形**と**色**の双方で意味を持ち、
  - **ライン完成**（行消し）
  - **同色連結4個以上**（塊消し／連鎖）
  のいずれでも盤面を消去できる。
- **狙い**：初心者はライン消しで爽快に、上級者は色塊連鎖と同時消し（ハイブリッド）で高得点を狙える設計。

## 2. 対象プラットフォーム
- iOS/Android スマホブラウザ（Safari/Chrome）
- 画面縦向き前提（片手操作最適化）。
- **PWA対応**（ホーム追加・オフライン・スプラッシュ）。

## 3. 基本ルール
### 3.1 フィールド
- グリッド：**幅 10、縦 20（天井 +2 の隠し行）**
- セル：空／色1〜色5（5色）／固定化ピース

### 3.2 ピース仕様
- 7種テトリミノ形（I,O,T,S,Z,J,L）を基本。
- 各セルが**色属性**を持つ（生成時に色を割り当て）。
- 供給：**7バッグ方式**（色は独立ランダム。連続同色偏り低減アルゴリズム採用）。

### 3.3 操作（タッチ）
- タップ：右回転（長押しで左回転）
- フリック左右：水平移動
- 下フリック：ソフトドロップ（×5落下速度）
- 上フリック：ハードドロップ（即着地）
- 画面下の**HOLDボタン**：1回/ピースの保持・交換
- 振動/SEは設定でON/OFF

### 3.4 落下・ロック
- 自然落下：レベルで速度上昇
- ロック遅延：500ms（移動/回転でリセット、上限 15回）
- SRS互換回転+壁蹴り（簡易）

### 3.5 消去判定（ハイブリッド）
1. **ライン消し**：行内が空きなしで消去。従来テトリス同等。
2. **色塊消し**：同色セルが**上下左右で4個以上**連結で消去。
3. **同時発生時**：
   - まず**ライン消し**を適用 → 盤面重力落下 →
   - 次に**色塊消し**を一括適用 → 落下 → 連鎖チェックを**色塊のみ**で継続（色塊連鎖）。
   - ラインは着地直後のみ判定（連鎖中は発生しない）。

### 3.6 連鎖
- 色塊の消去後の落下で新たな塊が4+になれば**連鎖+1**。
- 最大連鎖数は理論上無制限（実用は10前後）。

### 3.7 ガーシステム（対戦時）
- 攻撃量=（ライン数/同時色消し数/連鎖段数）から算出し、相手に**灰色おじゃま**を送る。
- おじゃまは次出現ピース着地後に下から出現。

## 4. スコア設計
- **ベース**
  - 1ライン：100
  - 2ライン：300
  - 3ライン：500
  - 4ライン：800
  - 色塊：1塊あたり `30 × (塊サイズ)`
- **ボーナス**
  - 同時消し（ライン+色塊）：×1.5
  - 連鎖：`×(1 + 0.5×n)`（n=連鎖段数）
  - ハードドロップ：距離×2、ソフトドロップ：距離×1
- **レベル補正**：`×(1 + level×0.05)`

## 5. ゲームモード
- **MARATHON**：レベル1開始、一定ライン消しでレベルUP、ゲームオーバーまでスコア。
- **TIME ATTACK**：2分。総スコア競争。
- **VS BOT**：AI難易度3段階。ガー適用。
- **LOCAL 2P（β）**：同端末交互/縦分割（回線対戦は将来）。

## 6. UI/レイアウト（縦画面）
- 上部：スコア／レベル／時間
- 中央：盤面（10:20を端末幅に応じてスケール）
- 右：NEXT（5個表示）／HOLD
- 下部：操作ガイド（透明）
- 設定：音量、バイブ、配色モード（色覚多様性対応）

## 7. アート&サウンド
- カラー：色覚対応パレット（5色）+模様差分（縞/ドット等）
- SE：回転、ロック、ライン、塊、連鎖、ドロップ
- BGM：アップテンポ（モード別2曲）

## 8. 難易度カーブ
- レベル1落下間隔：1000ms → レベル毎に -60ms（下限 120ms）
- おじゃま頻度（VS）：難易度依存

## 9. 生成アルゴリズム
- **形**：7バッグシャッフル
- **色**：直前同色3連以上を避ける重み付け乱数
- **初期穴**：無し（VSのみ相手から）

## 10. 主要システム設計
### 10.1 データ構造（例）
```ts
// Grid[y][x] = { color: 0..5, filled: boolean }
// color=0 は空

```

### 10.2 メインループ（擬似）
```txt
onTick -> 落下 or ソフト/ハード反映 -> ロック判定
着地 -> ライン判定 -> 消去 -> 落下
      -> 色塊探索(BFS/DFS) -> 4+ をマーキング -> 消去 -> 落下 -> 連鎖ループ
      -> 次ピース生成
```

### 10.3 色塊探索
- BFS/DFSで同色連結数カウント。閾値4以上を消去対象。
- 一斉マーキング→同時消し。

### 10.4 衝突/回転
- SRS準拠のオフセット表（簡易）で壁蹴り。

### 10.5 スコア計算
- 消去イベント単位でベース点→同時消し補正→連鎖補正→レベル補正。

# 追加：詳細仕様（実装に直結する数値・プロトコル）

## A. 画面・座標・タイムベース
- **座標系**：左上原点。1セル=**32px**（端末幅に応じて整数倍率スケール：×0.75, ×1, ×1.25, ×1.5）。
- **論理解像度**：盤面 10×22（うち 2 行隠し）。UIは相対レイアウト。
- **メインループ**：`requestAnimationFrame`。**ロジック固定Δt=16.666ms**（60FPS仮定、可変FPS時はアキュムレーター方式で複数tick処理）。
- **入力ポーリング**：毎フレーム収集、同フレーム内で優先度 `回転 > 左右移動 > ドロップ`。

## B. 入力・操作閾値
- **タップ回転**：タップ距離 < **12px** かつ **120ms–320ms** のタップを回転入力。
  - シングルタップ=右回転、**長押し>=400ms**=左回転。
- **フリック**：
  - 左右：水平移動 1 マス。閾値 **40px**、時間 **<250ms**。
  - 下：ソフトドロップ開始（押下中持続）。下方向 **40px** 閾値。
  - 上：ハードドロップ。上方向 **60px**、**<300ms**。
- **長押し移動（DAS/ARR）**：
  - **DAS=160ms**, **ARR=33ms**（DAS後は33msごとに1マス）。
- **HOLD**：1 ピースに 1 回。押下時点でキュー切替。着地後に回数リセット。
- **ヒットスロップ**：ボタンUIは見た目+**8px**の当たり判定拡張。

## C. ピース/生成/出現
- **出現位置**：`x=3, y=20`（4×4マトリクス左上座標）。Iミノのみ `x=3, y=20`、Oミノ `x=4, y=20` に補正。
- **回転系**：SRS（壁蹴りオフセットは付録E）。Z/J/L/S/T は共通表、I は別表。O は回転移動なし。
- **ロック遅延**：**500ms**。移動/回転でリセット。**最大リセット回数=15**。
- **ゴースト**：有効（設定で切替）。衝突判定で最下点を描画、入力には影響しない。
- **Next**：5個表示。**7バッグ**枯渇時に再シャッフル。
- **色割当**：各ピースの 4 セルに同一色を割当。色は 5 色から重み付けランダム（直近 3 ピース同色連続の確率を 20% に抑制）。

## D. 消去・重力・連鎖手順（厳密）
1. **着地確定** → **ライン判定**
2. ライン成立行を同時消去。消去演出 **150ms**。
3. **重力落下**（空セル詰め）→ 盤面安定。
4. **色塊探索**：上下左右連結で **>=4** セルの同色グループを同時マーキング。
5. 1 つ以上あれば同時消去（演出 **150ms**）→ **連鎖数+1**。
6. 重力落下 → 盤面安定 → 4 に戻る（色塊連鎖のみ継続）。
7. 4 で見つからなければ **連鎖終了** → 次ピース出現。
- **同時消しボーナス**はステップ2と5の**同一サイクル**でラインと色塊が両方起きたとき適用。

## E. スコア（厳密式）
- **ライン**：`L1=100, L2=300, L3=500, L4=800`
- **色塊**：グループごとに `30 × size`、同時に複数グループがあれば合算。
- **同時消し補正**：そのサイクルの小計に `×1.5`。
- **連鎖補正**：色連鎖段 `n` に対して `×(1 + 0.5×n)` をそのサイクルの小計に適用。
- **ドロップ**：ハード=到達距離×2、ソフト=落下距離×1 を着地時に加点。
- **レベル補正**：全てのサイクル小計に最後に `×(1 + 0.05×level)`。
- **例**：L2（300）+色塊(6個=180)、同時消し → (480×1.5)=720、連鎖2段目で (720×1.5)=1080、レベル3で×1.15 → 1242（端数四捨五入）。

## F. レベル/速度テーブル
- **開始レベル=1**、必要ライン：`10 + (level-1)×5` クリアで次レベル。
- **重力間隔(ms)**：`max(120, 1000 - 60×(level-1))`。
- **ソフトドロップ**：重力速度 **×5**。

## G. 対戦時のおじゃま計算（灰色）
- 攻撃ゲージ=各サイクルの**攻撃点**合算 → 行換算して送信（1 行= 1 段）。
- **攻撃点**：
  - L1=0.5, L2=1.5, L3=2.5, L4=4
  - 色塊：`(size-3) × 0.3`（4個で0.3、8個で1.5）
  - 連鎖ボーナス：`0.5 × n` 追加
  - 同時消し：+1
- **送信タイミング**：相手の次ピース**着地後**。受信待機は最大 **8 行**キュー。
- **相殺**：同フレームに攻撃成立で相殺。余剰のみ送信。

## H. 視認性・配色
- **色**（RGB）：
  - 赤(232, 85, 75) 斜線、青(70, 120, 240) ドット、緑(70, 190, 110) 市松、黄(245, 200, 70) 横線、紫(165, 95, 210) 砂目
- **コントラスト**：セル外枠 α=0.25、内部模様は 2px パターン。
- **おじゃま**：グレー(160,160,160) 模様なし。

## I. アニメーション・演出
- 消去フェード **150ms**、落下イージングなし（ステップ移動）。
- 連鎖表示：`n chain!` ポップ **600ms**。
- 画面シェイク：L4 または連鎖3+で **70ms** 微振動。
- サウンド：
  - `sfx_rotate.wav`, `sfx_lock.wav`, `sfx_line.wav`, `sfx_group.wav`, `sfx_chain.wav`, `sfx_drop.wav`
  - フォーマット：**.mp3 + .ogg** 同梱、**-6dB** 正規化。多重発音は **3 同時**上限。
